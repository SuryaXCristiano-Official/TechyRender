# TechyRender — Command Center

> **A self-hosted bot management dashboard + Render.com API controller.**  
> Single-file PHP frontend with a Python-powered bot backend. Deploy, monitor, and control your Render services and Telegram bots from one unified interface.

---

## Overview

TechyRender is a two-component system:

| Component | File | Stack | Role |
|-----------|------|-------|------|
| **Dashboard** | `index.php` | PHP + SQLite + SMTP | Web UI for managing bots & Render services |
| **Bot Engine** | `core_logic.py` → `TechyRender.py` | Python + MongoDB | REST API + heartbeat + web UI for each bot |

### How It Works

```
User Browser ──→ index.php (dashboard)
                      │
                      ├── PHP proxies API calls to Render.com
                      ├── PHP proxies API calls to your bots
                      └── Manages users via SQLite
                      
Your Bot ──→ TechyRender.py (runs alongside your bot)
                 │
                 ├── Serves /getstatus, /connectweb, /unlink, /update
                 ├── Background heartbeat (ping + DB sync)
                 └── Web UI at RENDER_EXTERNAL_URL
```

## Features

### Dashboard (`index.php`)
- **Bot Fleet Management** — add, list, delete bots with auto-detected names
- **Render.com API Control** — restart, suspend, resume, force deploy, manage env vars
- **TechyRender API** — link/unlink TR ID, update ping interval, get bot status
- **User System** — signup/login with email OTP verification via SMTP
- **Admin Panel** — user registry showing all registered users and their bot counts
- **Live Terminal** — styled output log for each selected bot
- **Responsive Design** — works on desktop and mobile

### Bot Engine (`TechyRender.py`)
- **REST API** — `/getstatus`, `/connectweb/techyrender`, `/techyrender/unlink`, `/techyrender/update`
- **Web UI** — status page matching the dashboard design language
- **MongoDB Sync** — state persistence across restarts
- **Heartbeat** — periodic ping to keep Render service alive
- **Suspension Detection** — auto-shutdown when suspended via dashboard
- **Bot Process Management** — auto-restart on crash

## Quick Start

### 1. Deploy the Dashboard (`index.php`)

**Requirements:** PHP 8.0+, SQLite3, PDO, cURL, OpenSSL

```bash
# Place index.php in your web root
# Ensure PHP has write permissions for SQLite database
php -S localhost:8000 index.php
```

The database (`techyrender.sqlite`) is auto-created on first run.  
The admin account is seeded from the constants at the top of `index.php`:

```php
define('ADMIN_EMAIL', 'your@email.com');
define('ADMIN_PASS',  'your-password');
define('ADMIN_UNAME', 'YourUsername');
```

**SMTP Configuration** (for OTP emails):

```php
define('SMTP_HOST', 'smtp.gmail.com');
define('SMTP_PORT', 587);
define('SMTP_USER', 'your@email.com');
define('SMTP_PASS', 'your-app-password');
```

> **Note:** Use a Gmail App Password (not your regular password) for SMTP.

### 2. Deploy the Bot Engine (`TechyRender.py`)

**Requirements:** Python 3.8+, `requests`, `python-dotenv`, `pymongo` (optional)

```bash
# Install dependencies
pip install requests python-dotenv pymongo cryptography

# Set up environment
cp .env.example .env   # or set variables manually

# Run the engine
python TechyRender.py
```

**Environment Variables (`.env`):**

| Variable | Required | Default | Description |
|----------|----------|---------|-------------|
| `BOT_NAME` | No | — | Display name on the bot's web UI |
| `BOT_DEV` | No | — | Developer credit on the bot's web UI |
| `RENDER_SERVICE_NAME` | No | `BOT_NAME` or `TechyRender-Bot` | Used as MongoDB document key |
| `RENDER_EXTERNAL_URL` | No | — | Your bot's public URL (for heartbeat ping) |
| `PORT` | No | `8080` | HTTP server port |
| `MONGO_URI` | No | — | MongoDB connection string (for state persistence) |
| `BOT_FILENAME` | No | `Test.py` | Your bot script to run alongside |

### 3. Build from Source (Optional)

```bash
python encrypt.py
# Produces TechyRender.py from core_logic.py
```

## Connecting a Bot to the Dashboard

### Step 1: Add a Bot in the Dashboard

1. Log into the dashboard
2. Click **"+ ADD BOT"**
3. Enter:
   - **Render API Key** — your Render.com API key (`rnd_...`)
   - **Render Service ID** — your service ID (`srv-...`)
   - **Bot URL** — your bot's public URL (`https://your-bot.onrender.com`)
4. The bot name is auto-detected from `/getstatus`

### Step 2: Link Your TR ID

1. Select the bot card in the dashboard
2. Click **"Link TR ID"** in the Bot Controls section
3. Your TechyRender ID is sent to the bot automatically
4. The bot's "Account" status changes to **Connected**

### Step 3: Control Your Bot

Once linked, you can:
- **Get Status** — view real-time bot metrics
- **Update Interval** — change the heartbeat ping frequency
- **Unlink TR ID** — disconnect from the dashboard
- **Restart / Suspend / Resume** — control the Render service
- **Force Deploy** — trigger a new deployment with optional cache clear
- **Env Variables** — add/edit/remove environment variables on Render

## API Reference (Bot Engine)

The bot engine at `RENDER_EXTERNAL_URL` exposes these endpoints:

### `GET /getstatus`
Returns bot state as JSON.

### `GET /connectweb/techyrender?id={TR_ID}`
Links the bot to a TechyRender account.

### `GET /techyrender/unlink?id={TR_ID}`
Unlinks the bot from its TechyRender account.

### `GET /techyrender/update?id={TR_ID}&interval={SECONDS}`
Updates the heartbeat ping interval.

## Design

- **Dark AMOLED theme** — pure black backgrounds (`#000`) for efficiency on OLED screens
- **GPU-composited animations** — all hover effects use `transform` (no layout thrashing)
- **No `backdrop-filter` on sticky elements** — avoids scroll performance issues
- **Explicit CSS transitions** — no `transition: all` anywhere
- **Minimal continuous animations** — only 3 infinite animations, rest are one-shot entrances

## Security

- CSRF protection on all state-changing actions
- Rate limiting on auth and OTP endpoints
- Input validation and sanitization
- Internal URL blocking (SSRF prevention)
- Session-based authentication with OTP verification
- Tamper detection in the built `TechyRender.py`
